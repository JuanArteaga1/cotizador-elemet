MÓDULO: BACKEND Y API REST
----------------------------

US-41 | Backend Node.js + Express
  Como: desarrollador
  Quiero: un backend robusto con API REST
  Para: persistir datos de usuarios, cotizaciones y configuración en base de datos
  Criterios:
    - Stack: Node.js + Express + JavaScript + Prisma ORM + SQLite/PostgreSQL
    - Arquitectura en capas: routes, middleware, lib, utils
    - CORS habilitado para comunicación con frontend
    - Variables de entorno configurables (.env)
    - Servidor escucha en puerto configurable (por defecto 3001)

US-42 | API de autenticación con JWT
  Como: usuario
  Quiero: registrarme e iniciar sesión de forma segura
  Para: que mis datos estén protegidos y solo yo pueda acceder a ellos
  Criterios:
    - POST /api/auth/register - crea usuario con bcrypt
    - POST /api/auth/login - valida credenciales y retorna JWT
    - GET /api/auth/me - obtiene datos del usuario autenticado
    - Passwords hasheados con bcrypt (10 rounds)
    - Tokens JWT con expiración de 7 días
    - Respuestas JSON estandarizadas con { user, token }

US-43 | API de cotizaciones y configuración (aislamiento por usuario)
  Como: usuario
  Quiero: que mis cotizaciones y configuraciones sean privadas
  Para: que ningún otro usuario pueda ver o modificar mis datos
  Criterios:
    - Todas las rutas de quotes y config requieren autenticación JWT
    - GET /api/quotes - lista solo cotizaciones del usuario autenticado
    - POST /api/quotes - crea cotización asociada al usuario
    - PUT /api/quotes/:id - actualiza solo si pertenece al usuario
    - DELETE /api/quotes/:id - elimina solo si pertenece al usuario
    - GET /api/config - obtiene configuración del usuario (o crea default)
    - PUT /api/config - actualiza configuración del usuario
    - Campos JSON almacenados como string en la base de datos
    - Middleware authMiddleware verifica JWT en cada petición protegida

US-44 | Sincronización frontend-backend
  Como: usuario
  Quiero: que mis datos se guarden automáticamente en el servidor
  Para: no perder información si cambio de dispositivo o borro el navegador
  Criterios:
    - Al iniciar sesión: se cargan quotes y config desde el backend
    - Al crear/editar/eliminar quote: se sincroniza con backend
    - Al actualizar config: se sincroniza con backend
    - API service centralizado en frontend (api.ts)
    - Token JWT enviado en header Authorization Bearer
    - Fallback a datos locales si el backend no responde

MÓDULO: COTIZACIÓN - WIZARD 5 PASOS
-------------------------------------

US-05 | Paso 1 - Datos del cliente
  Como: usuario
  Quiero: ingresar nombre del cliente y nombre del proyecto
  Para: identificar cada cotización
  Criterios:
    - Campo "Nombre completo" (requerido)
    - Campo "Proyecto" (requerido)
    - Botón "CONTINUAR →"
    - Validación: muestra advertencia si campos están vacíos
    - Indicador de progreso: "Paso 1 de 5"

US-06 | Paso 2 - Área del terreno
  Como: usuario
  Quiero: ingresar las dimensiones del terreno de dos formas posibles
  Para: calcular el área de construcción
  Criterios:
    - Modo "Dimensiones": ingresar medidas de lote rectangular o irregular
    - Modo "Área Directa": ingresar área del primer piso directamente
    - Diagrama SVG interactivo con inputs embebidos para frontal, posterior, laterales
    - Selector de forma del lote: Rectangular / Irregular
    - Slider de índice de ocupación (50%-100%)
    - Visualización en tiempo real: área del lote, área primer piso
    - Soporte para forma irregular (frontal ≠ posterior, lateral izq ≠ lateral der)

US-07 | Paso 3 - Volados y pisos múltiples
  Como: usuario
  Quiero: configurar el número de pisos y áreas de voladizo
  Para: calcular el área total construida en edificios de varios pisos
  Criterios:
    - Selector de número de pisos (1-10)
    - Si >1 piso: campo de tamaño de volado (metros)
    - Checkboxes para seleccionar fachadas con volado: Frontal, Posterior, Lateral Izq, Lateral Der
    - Cálculo automático: área de volado por piso, área total piso superior
    - Visualización del área total estimada

US-08 | Paso 4 - Servicios
  Como: usuario
  Quiero: seleccionar los servicios de diseño que incluiré en la cotización
  Para: calcular el valor total del proyecto
  Criterios:
    - Opción "Paquete Completo Premium" (todo incluido, precio fijo por m²)
    - Si NO es paquete completo: sub-paquetes disponibles
    - Servicios individuales con precio por m²
    - Al seleccionar sub-paquete "Instalaciones": ocultar servicios eléctrico e hidrosanitario
    - Visualización del valor total en tiempo real
    - Check marks visuales en servicios seleccionados
    - Servicios disponibles: Diseño Arquitectónico, Estructural, Hidrosanitario,
      Eléctrico, Renders 3D, Recorrido 3D, Presupuesto, Licencias

US-09 | Paso 5 - Resumen y ajuste final
  Como: usuario
  Quiero: revisar el resumen de la cotización antes de guardarla
  Para: aplicar descuentos, agregar servicios adicionales y ver el plan de pagos
  Criterios:
    - Datos del cliente y proyecto
    - Sección de "Servicios Adicionales" con formulario inline
    - Campos para agregar: nombre del servicio + valor
    - Lista de servicios adicionales agregados con opción de eliminar
    - Campo de descuento (ajuste de valor)
    - Valor base, descuento y valor final mostrados
    - Plan de pagos con desglose por cuota
    - Botón "GUARDAR" que persiste la cotización
